The Problem

Managing AI APIs across a team shouldn't feel like this.

Without a Gateway

  • API keys scattered across apps, .env files, and Slack messages
  • No idea who spent $400 on GPT-4o last Tuesday
  • Switching providers means rewriting every integration
  • One leaked key exposes your entire account
  • Budget surprises at the end of every month

With TokenParty

  • One endpoint, scoped keys per user with provider-level access control
  • Real-time cost dashboard — by user, model, provider, and tag
  • Switch providers in config. Your apps never change.
  • Isolated tokens — revoke one without touching the rest
  • Monthly budgets with automatic enforcement

Get Started

Running in under a minute.

Install

One command. No database setup, no Redis, no external services.

# via npm
npm install -g @tokenparty/tokenparty

# or Docker
docker compose up -d

Configure

Add your providers and create user tokens. Config hot-reloads on save.

# Open the dashboard
open http://localhost:3456

# Add providers, create tokens,
# set budgets — all from the UI

Use

Point any SDK at your gateway. That's it.

# Works with any OpenAI SDK
client = OpenAI(
  base_url="http://localhost:3456/v1",
  api_key="tp-your-token"
)
Everything Included

No plugins. No add-ons. No surprises.

Protocol Translation

OpenAI ↔ Anthropic format conversion. Use any SDK with any provider.

SSE Streaming

Full streaming with transparent protocol conversion between formats.

Smart Routing

Automatically picks the cheapest provider for each model request.

Budget Enforcement
